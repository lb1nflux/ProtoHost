**Project**: ProtoHost  
**Branch**: `main`  
**Description**: 原型托管平台。支持上传包含 HTML 原型的 ZIP 文件，并通过公开或密码保护的链接分享。

## 2. 详细需求说明

### US-001: 数据库架构搭建
- **描述**: 作为系统，我需要一个数据库来存储用户信息和项目元数据。
- **验收标准**:
  - [x] 创建 `users` 表，包含 email 和 password_hash。
  - [x] 创建 `projects` 表，包含 user_id, share_slug, 和 storage_path。
  - [x] 实现用户与项目之间的外键约束。
  - [x] 数据库支持 UTF-8 字符编码。

### US-002: JWT 后端认证系统
- **描述**: 作为用户，我希望能够安全地注册和登录。
- **验收标准**:
  - [x] 实现 AuthController，提供 `/api/auth/register` 和 `/api/auth/login`。
  - [x] 配置 Spring Security 并集成 JwtFilter。
  - [x] 实现 JwtUtil 用于 Token 的生成与校验。
  - [x] 确保 `/api/projects/**` 接口仅限认证用户访问。

### US-003: 原型上传与解压逻辑
- **描述**: 作为用户，我希望上传 ZIP 文件后系统能自动解压。
- **验收标准**:
  - [x] 实现 ProjectController 处理文件上传。
  - [x] 创建 ZipUtil，支持 GBK 和 UTF-8 编码的解压，解决乱码问题。
  - [x] 将解压后的文件存储在指定目录 (`app.upload.base-path`)。
  - [x] 将项目元数据（名称、入口文件、Slug）保存至数据库。

### US-004: 分享链接与密码保护
- **描述**: 作为所有者，我希望通过 Slug 分享项目，并可选设置访问密码。
- **验收标准**:
  - [x] 为每个项目生成唯一的 32 位 `share_slug`。
  - [x] 实现 `is_public` 和 `access_password_hash` 逻辑。
  - [x] 创建 ShareController，允许通过 Slug 获取项目详情（无需全局认证）。

### US-005: 前端认证与状态管理
- **描述**: 作为用户，我需要一个功能完整的界面来管理我的会话。
- **验收标准**:
  - [x] 在 `frontend/src/stores/auth.ts` 中设置 Pinia 认证仓库。
  - [x] 实现登录页 (`LoginPage.vue`) 和注册页 (`RegisterPage.vue`)。
  - [x] 配置 Axios 拦截器，自动在请求头中附加 JWT Token。
  - [x] 实现路由守卫，拦截未认证的访问。

### US-006: 控制台与上传界面
- **描述**: 作为用户，我希望查看已上传的项目并上传新项目。
- **验收标准**:
  - [x] 实现 `DashboardPage.vue`，以卡片形式展示项目列表。
  - [x] 实现 `UploadPage.vue`，包含文件选择和元数据表单。
  - [x] 提供上传进度提示及成功/错误通知。
  - [x] 允许用户查看项目状态（公开/私有）。

### US-007: 原型预览系统与访问控制
- **描述**: 作为访问者，我希望在通过身份验证（如有必要）后通过 iframe 预览原型。
- **验收标准**:
  - [x] 实现 `ViewPage.vue` 处理公开/私有访问逻辑。
  - [x] 使用指向解压后入口文件的 iframe 渲染原型。
  - [x] 如果项目受密码保护，提示用户输入密码。
  - [x] 支持 iframe 内 CSS/JS 等资源的正确加载。
